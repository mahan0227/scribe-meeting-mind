Nineteen opinionated AI microtools — each its own repo

Neuron Suite is nineteen separate Next.js apps (one repo each) for real work: prompts, meetings, grounded docs, code review, brand, data stories, customer voice, learning paths, and more. You bring your OpenAI API key; deploy on your infra. No generic chat wrapper—each tool does one job with structured JSON you can trust in workflows.

What is Neuron Suite?

Neuron Suite is a family of nineteen small web applications, each in its own repository, built the same way: Next.js 16, React 19, Tailwind v4, and OpenAI chat completions with JSON-shaped outputs. There is no shared “mega product”—only sharp tools you can deploy, fork, or remix independently.

Every app is BYOK (bring your own key): users paste an OpenAI API key in the browser; your deployment never needs a platform master key. That keeps cost, privacy, and compliance under your account—ideal for internal operators, agencies, and builders who outgrew generic chat wrappers.

Together the suite covers prompt engineering, meetings, grounded documents, code & security review, brand & voice, data storytelling, customer voice, learning paths, and more—each tool documented with what it is, why it is useful, and where teams use it in its README.

Tool grid

Each card summarizes what the tool is, why teams reach for it, and where it usually lives (roles and workflows). Crop cards for Instagram; print to PDF for LinkedIn leave-behinds.

Neuron Prompt Forge

Single-feature app: goal → structured prompt pack JSON (primary, variants, checklist, risks, system hint).

Why it’s useful

  • Aligns PM + eng on “done” for a prompt before shipping to code.
  • Surfaces failure modes early for bots, RAG, and internal copilots.

Where it’s used

  • Product, support automation, eval prep, agency workshops.

Echo Meeting Mind

Transcript → fast operational minutes JSON: decisions, owners, risks, parking lot.

Why it’s useful

  • Turns messy calls into trackers without a full-time note taker.
  • Separates real decisions from discussion noise.

Where it’s used

  • Eng managers, startups, sales QBRs, remote standups.

Scribe Meeting Mind

Same transcript input, richer output: board-style narrative, dissent-aware decisions, mitigations, next agenda.

Why it’s useful

  • Produces governance-grade artifacts for execs and committees.
  • Preserves why a decision happened—not just bullet titles.

Where it’s used

  • Boards, incident command, program steering, audit-heavy orgs.

Atlas Doc Mind

Evidence-first Q&A over one pasted document with citations, confidence, and follow-ups.

Why it’s useful

  • Reduces hallucinated “facts” in internal knowledge workflows.
  • Makes answers checkable with short verbatim spans.

Where it’s used

  • Legal/procurement triage, RFC comprehension, finance memos, policy desks.

Aura Brand Alchemy

Voice samples + deliverable brief → voice DNA, lexicon, on-brand asset in Markdown, rationale JSON.

Why it’s useful

  • Scales founder voice without dilution across freelancers.
  • Explains creative choices for stakeholder review.

Where it’s used

  • B2B SaaS launches, agencies, employer brand, indie maker social.

Prism Code Sentinel

Security- and correctness-biased review JSON: severities, exploit sketches, tests, refactors.

Why it’s useful

  • Compresses “what to look at first” before human or SAST review.
  • Connects findings to concrete tests.

Where it’s used

  • Squads without full-time appsec, OSS triage, pre-audit sweeps.

Lens Vision Brief

Reference image + creative intent → creative-direction JSON (palette, composition, hooks, don’ts).

Why it’s useful

  • Aligns marketing and design before expensive shoots.
  • Surfaces clichés to avoid early.

Where it’s used

  • Growth teams, agencies, landing experiments, key art exploration.

Crystal JSON Oracle

Plain-language data intent → JSON Schema draft, examples, migration notes JSON.

Why it’s useful

  • Shortens product language → typed contract gap.
  • Seeds fixtures for tests and partner docs.

Where it’s used

  • Events/webhooks, ETL contracts, mobile payloads, B2B onboarding.

Glyph Regex Whisperer

Natural language + flavor → regex, tests, caveats JSON for safer patterns.

Why it’s useful

  • Saves regex rabbit holes for one-off parsers and filters.
  • Flags foot-guns like catastrophic backtracking when relevant.

Where it’s used

  • SRE log routing, CSV hygiene, editor searches, lightweight validators.

Arena Dialectic

Two personas debate a tension with steel-manning; JSON includes tradeoffs, path, experiments.

Why it’s useful

  • Stress-tests decisions before irreversible spend.
  • Ends with experiments, not vibes.

Where it’s used

  • Architecture reviews, roadmap bets, leadership offsites, RFC prep.

Lumen Learn Path

Topic + time budget → weekly curriculum JSON with drills, resources, quizzes, anti-patterns.

Why it’s useful

  • Realistic adult learning plans—not generic blog outlines.
  • Checkpoints with rubrics keep progress measurable.

Where it’s used

  • Engineering upskilling, career switches, manager growth plans.

Semantic Bisect

Two texts + optional lens → meaning-level diff: divergences, assumptions, merge playbook, bridge paragraph.

Why it’s useful

  • Surfaces silent disagreements before expensive meetings.
  • Produces neutral language both sides can sign.

Where it’s used

  • PRD vs eng spec, policy versions, partnership redlines, research memos.

Steel Socratic

Belief or bet → Socratic ladder JSON with cruxes and “how to update your mind” policy.

Why it’s useful

  • Improves epistemic hygiene without moralizing.
  • Makes testable cruxes explicit for experiments.

Where it’s used

  • Founders, investors, policy exploration, journaling, research writing.

Persona Resonance

Persona brief + draft copy → resonance score, drift, rewrites, micro style guide JSON.

Why it’s useful

  • Quantifies voice alignment for distributed writers.
  • Extracts do/don’t lists from the brief itself.

Where it’s used

  • Brand, games narrative, support macros, exec ghostwriting QA.

Plain Bind Clarity

Dense clauses → plain obligations, risks, glossary, checklist JSON—not legal advice.

Why it’s useful

  • Speeds first-pass comprehension for business stakeholders.
  • Flags ambiguities to resolve with counsel.

Where it’s used

  • MSAs/DPAs under counsel oversight, HR handbooks, procurement SLAs.

Prism Data Story

Metrics/CSV + audience → narrative JSON with chart ideas, anomalies, ethics limits, social snippets.

Why it’s useful

  • Aligns data and story without overclaiming from thin slices.
  • Multi-channel copy from one analysis pass.

Where it’s used

  • Board decks, investor updates, growth reviews, impact reporting.

Arc Code Narrative

Code snippet → systems story JSON: cast, acts, villains, glossary, onboarding reading order.

Why it’s useful

  • Accelerates onboarding when READMEs lag reality.
  • Memorable mental model for talks and internal docs.

Where it’s used

  • New hire ramp, OSS contributor guides, diligence skim, teaching.

Forge Prompt Atelier

Mission + stack + safety → full prompt kit JSON: templates, few-shots, eval rubric, negatives, ops notes.

Why it’s useful

  • Productionizes prompts beyond a single clever string.
  • Supports audits and regression thinking.

Where it’s used

  • ML platforms, safety review, enterprise capability versioning.

Echo Voice of Customer

Qualitative feedback blob → themes with quotes, roadmap bets, messaging guardrails JSON.

Why it’s useful

  • Anchors prioritization in evidence, not HIPPO opinion.
  • Bridges PM, marketing, and support on customer language.

Where it’s used

  • Quarterly planning, UX synthesis, positioning refresh, early PMF.
